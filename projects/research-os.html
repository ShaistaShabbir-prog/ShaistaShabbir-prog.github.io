ResearchOS — AI Peer Review Platform

How I built a Claude-powered research paper analysis system from scratch

Problem

Peer review quality varies widely. Reviewers miss reproducibility issues, statistical errors, and unsupported claims. Area chairs have no tools to quickly synthesise conflicting reviews. The process is manual, slow, and inconsistent.

Solution

An AI-powered platform that analyses papers across 6 dimensions: claims validation, reproducibility scoring, citation checking, reviewer fatigue detection, semantic similarity, and structured badge generation. Claude API provides the AI layer; heuristic fallbacks ensure the system works without an API key.

Architecture

  • Frontend: Next.js 14, TypeScript, Tailwind CSS → Vercel
  • Backend: FastAPI (Python), SQLite, 29 REST endpoints → Render.com
  • AI: Anthropic Claude API (claude-haiku for fast tasks, claude-sonnet for deep review)
  • Auth: JWT (HMAC-SHA256), PBKDF2 passwords, API key system
  • Tests: 247+ tests, GitHub Actions CI

Lessons learned

  • Heuristic fallbacks are essential — AI APIs have rate limits and costs
  • JWT without external libraries reduces dependencies and attack surface
  • Streaming responses dramatically improve perceived performance for long analyses
  • Type-safe API client (TypeScript) catches bugs before runtime
